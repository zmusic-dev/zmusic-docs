## 再生時に音が出ない

もし`KCaldron`を使用している場合、申し訳ありませんが、このサーバーはサポートしていません。

しかし、どうしても Mod サーバーを使用したい場合は、2つの解決策があります：

- [BungeeCord](https://www.spigotmc.org/wiki/bungeecord) または [WaterFall](https://papermc.io/downloads#Waterfall)
  などのプロキシサーバーを使用する。
- Arclight, LoliServer, Mohist, Uranium など、このプラグインを**サポートする可能性がある** Mod サーバーコアを使用する。

もし **Spigot, Paper, Yatopia, Sugarcane などのバニラプラグインサーバーコア**を使用している場合、以下の条件を満たしているか確認してください：

- クライアントに ZMusic Mod（1.7.10, 1.12 以降のバージョン）または AudioBuffer Mod（1.12 以下のバージョン）がインストールされている
- サーバーに ZMusic プラグインが正常にインストールされている
- サーバーのプラグインフォルダにインストールされているのは ZMusic シリーズのプラグインであり、ZMusic Mod ではない（Mod はクライアントにインストールする必要があります）
- クライアントは Fabric または Forge であり、**バニラではなく**、AudioBuffer **または** ZMusic Mod がインストールされている
- ネットワーク環境が良好である
- リクエスト時に使用しているプラットフォームは NetEase Cloud Music です！！！

もし上記の条件のいずれかを満たしていない場合は、さらに調整を行ってください。

もちろん、すべての条件を満たしているのに音楽が再生されない場合は、外部 API を設定せずに **NetEase Cloud Music** を使用して再生することをお勧めします。

実際、自分で所有しているサーバーでホストすることをお勧めします。

## サーバーの読み込みが成功した後、`/zm` 関連のコマンドを使用すると、「エラー：プラグインの読み込み完了をお待ちください」と表示される

この問題が発生した場合は、ZMusic プラグインを最新バージョンにアップグレードすれば問題が解決します。

## プラグインが読み込めず、「AllMusic/AudioBuffer プラグインをアンインストールしてください」と表示される

サーバーのプラグインフォルダから AllMusic/AudioBuffer プラグインを削除してください。<br>
新しいバージョンの ZMusic では、BossBarAPI, ActionBarAPI 依存プラグインと AudioBuffer 前提プラグインを廃止しました。サーバーの Mod 通信チャンネルの競合を防ぐために、この措置を取らざるを得ませんでした。

## 音楽の再生に成功した後、クライアントに時間がかかると表示される（しかし実際は短い）

この問題の原因は、同じリクエストボタンを連続してクリックする間隔が短すぎる可能性があります。内部スレッドの呼び出しが処理しきれず、間違った時間が表示されていますが、実際の使用には影響しません。

## NetEase Cloud Music にログインする際、`java.net.UnknownHostException` 例外が発生する

設定ファイルで外部 API を設定する際にドメインが間違っています。ドメインが有効か確認してください。<br>
ドメインが有効な場合は、サーバーの DNS が正常か、または DNS キャッシュをフラッシュしてください。<br>
レンタルサーバーのユーザーは通常このような問題は発生しません。

## プラグインの読み込み時、`java.lang.NoClassDefFoundError com/google/gson/xxx` エラーが発生する

サーバーに gson ライブラリが組み込まれていません。gson ライブラリを含むプラグインをインストールしてください。（まさか、gson をサポートしていないサーバーがあるなんて！）

## リクエスト成功後、コンソールに黄色いエラーが大量に表示される

2.5 から、旧版の進行状況呼び出し方法は廃止されました。最新バージョンに更新してください。

## ZMusic をインストールした後、クライアントが起動しない (Fabric)

- インストールした Mod のバージョンが正しいか確認してください。例えば、クライアントが Fabric Mod ローダーを使用しているのに、`zmusic-forge-X.X.X-X.X.X.jar` をダウンロードした場合<br>
- Fabric バージョンの Mod をダウンロードした場合、`Fabric-API` 前提 Mod がインストールされているか確認してください<br>
- この Mod は `CardBoard` と互換性がありません

## プラグインの読み込み後にエラーが表示される：「エラー：プラグインの読み込み完了をお待ちください」

この問題が発生した場合、以下の状況を確認してください。

- インストールしているプラグインのバージョンが最新か確認する。
- `ZMusic` プラグインフォルダに書き込み権限があるか確認する（特に Linux ベースのシステムでは、 ~~chmod 777 を強行することをお勧めします~~ 実際、777 を設定するのは非常に危険な行為です）
- サーバーが非同期スレッドをサポートしているか確認する（一般的に `Bukkit` ベースのものはサポートしています。~~Mohist について聞かないでください！~~）
- プラグインフォルダを手動で修復する。`ZMusic` フォルダに `language` というフォルダを作成し、私たちのリポジトリの `master` ブランチから `language` フォルダを見つけ、`zh_cn.json` ファイルをダウンロードして `language` フォルダに入れ、サーバーをリロードまたは再起動する。

## `1.7.10` はサポートされていますか？

`1.7.10` バージョンの使用はお勧めしません。使用する場合は、以下の説明を確認してください。

- `Mohist/Uranium` サーバーのみサポート
- `Uranium` は `dev-4-b210` でのみテスト成功
- `Mohist` は `1.7.10-42` でのみテスト成功
- `チャットメッセージ` 以外、他のすべての歌詞表示は無効（Uranium を除く）
- `Uranium` 対応の Mod を使用すると、Title/ActionBar 表示が可能
- `1.7.10` バージョンに関する問題のフィードバックは受け付けません

## プラグインのコマンドが DeluxeMenu/TrMenu などのメニュープラグインから呼び出せない (BungeeCord/WaterFall)

これはプロキシサーバーの一般的な問題です。ZMusic のコマンドはプロキシサーバーコアに登録されていますが、プラグインは Spigot/Paper 側のコマンドしか呼び出せません。<br>
ここで折衷案を考えました。ユーザーに自分で入力させる方法です。ユーザーにとってはあまり親切ではありませんが、自分でやればこそ！

## [zmusic プラグインが常に zmusic.use 権限がないと表示され、権限グループに追加したことを確認している](https://www.mcbbs.net/forum.php?mod=viewthread&tid=1310665)

BungeeCord/Waterfall を使用している場合は、プロキシサーバーのプラグインディレクトリに LuckPerms-Bungee プラグインを追加し、`/lpb` コマンドで権限を付与してください。

## [[Sponge]サーバー ZMusic オンライン音楽が読み込まれない](https://www.mcbbs.net/forum.php?mod=viewthread&tid=1048579)

このプラグインは Sponge プラットフォームをサポートしていません。3.0 のリリースをお待ちください。（現在の 3.0 ベータ版は Sponge を半サポートしています）
